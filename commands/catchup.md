# Catch Up

You are resuming a research session. Do the following in order:

1. Read CLAUDE.md to load project context, environment, and conventions.

2. Read PROGRESS.md to understand:
   - What was accomplished in the last session
   - Where work stopped
   - What the next step is
   - Any open questions or blockers that were noted

3. Briefly confirm what you've loaded by summarizing in 3-5 bullet points:
   - Current project status
   - What was last worked on
   - What the immediate next step is
   - Any blockers or open questions to resolve
   - Any relevant files to be aware of

4. Produce an explicit TODO list derived from PROGRESS.md open questions, incomplete items, and next steps.
   Format it as a numbered list, ordered by priority (immediate next step first, longer-term items last):

   TODO:
   1. [Immediate next step from last session]
   2. [Any incomplete or in-progress items]
   3. [Open questions to resolve]
   4. [Longer-term items noted]

   If TODO.md exists in the project root, read it and merge its items with those from PROGRESS.md, deduplicating where possible. After confirming direction, write the merged TODO list back to TODO.md.

5. Ask: "Ready to continue — shall I start on [next step], or is there something else you'd like to prioritize?"

Do not begin any work or read any other files until the user confirms the direction.
