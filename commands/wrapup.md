# Wrap Up

You are closing out the current research session. Do the 
following in order:

1. Read PROGRESS.md to understand the previous state so 
   you can append to it correctly rather than overwrite it.

2. Read TODOS.md if it exists, to understand the current 
   task state.

3. Review the current conversation to identify:
   - What was accomplished this session
   - What files were created or modified
   - Any decisions that were made and why
   - What was left incomplete or partially done
   - What the logical next step is
   - Any open questions, blockers, or things to watch out for

4. Update PROGRESS.md by prepending a new session entry 
   at the TOP of the file (most recent first) in this format:

---
## Session: [DATE]

### Accomplished
- [bullet list of what was done]

### Files Changed
- [file path] — [brief description of change]

### Decisions Made
- [any architectural, analytical, or writing decisions 
   made this session and the reasoning]

### In Progress / Incomplete
- [anything started but not finished]
- [any todos added to TODOS.md this session]

### Next Step
[Single clear sentence describing exactly where to pick up]

### Open Questions / Watch Out For
- [anything unresolved or that needs attention next session]
---

5. Update TODOS.md to reflect session outcomes:
   - Mark any todos completed this session as complete
   - Add any new incomplete items or open questions that 
     emerged this session as new Active todos
   - Tag new todos with [from session DATE] for traceability
   - Do NOT remove or modify todos unrelated to this session

6. Confirm both files were updated. Display the "Next Step" 
   line and the current TODOS.md Active list so they are 
   visible before closing.

Do not summarize the entire project history — only this 
session. Keep each entry concise enough to be scannable 
in 30 seconds.
